## 1) Alcance

Este RFC consolida las decisiones implementadas para:

- **UC-WPL-03**: asignación semanal por empleado con validación por lote.
- **UC-WPL-04**: soporte de edición/validación para grilla semanal.
- Cobertura operativa por franjas de **5 minutos** (`0..287` slots por día).

También documenta la unificación de migraciones `ALTER` dentro de sus migraciones `CREATE` correspondientes.

---

## 2) Decisiones técnicas aprobadas

1. **Estrategia de PK/FK**
   - Se estandariza en `bigint` (`bigIncrements` / `unsignedBigInteger`) para las tablas de Scheduling.
   - `schedules.id` queda en `bigint` para compatibilidad con `weekly_schedule_assignments.schedule_id`.

2. **No solapamiento semanal (PostgreSQL)**
   - `weekly_schedules` aplica exclusión por rango de fechas:
   - `EXCLUDE USING gist (daterange(start_date, end_date, '[]') WITH &&)`.

3. **Granularidad operativa de 5 minutos**
   - `shift_activities.start_slot` y `end_slot` usan slots `0..288`.
   - Restricción: `start_slot < end_slot` y límites válidos por `CHECK`.

4. **Cobertura mínima por equipo/día/slot**
   - `coverage_requirements` define mínimos requeridos.
   - `coverage_snapshots` persiste el estado calculado (`assigned_count`, `required_min`, `deficit`).

## 3) Esquema final incorporado

### 3.1 Tablas base de planificación semanal

- `weekly_schedules`
  - `id`, `name`, `start_date`, `end_date`, `status`, timestamps.
  - `UNIQUE(name)`.
  - Constraint de no solapamiento en PostgreSQL.

- `weekly_schedule_assignments`
  - `weekly_schedule_id`, `employee_id`, `schedule_id`, `assignment_date`, `is_manual`.
  - `UNIQUE(weekly_schedule_id, employee_id, assignment_date)`.

### 3.2 Tablas de turno y actividades

- `shifts`
  - `weekly_schedule_assignment_id`, `employee_id`, `date`, `start_time`, `end_time`, `status`.
  - `UNIQUE(employee_id, date)`.
  - Índices: `shifts_employee_idx`, `shifts_date_idx`.

- `shift_activities`
  - `shift_id`, `activity_type`, `start_slot`, `end_slot`, `description`.
  - Índices: `shift_activities_shift_idx`, `shift_activities_slot_idx`.
  - `CHECK`: `start_slot >= 0 AND end_slot <= 288 AND start_slot < end_slot`.

### 3.3 Tablas de cobertura

- `coverage_requirements`
  - `team_id`, `date`, `slot_index`, `required_min`.
  - `UNIQUE(team_id, date, slot_index)`.
  - Índice: `coverage_requirements_team_date_idx`.

- `coverage_snapshots`
  - `team_id`, `date`, `slot_index`, `assigned_count`, `required_min`, `deficit`.
  - `UNIQUE(team_id, date, slot_index)`.
  - Índice: `coverage_snapshots_team_date_idx`.

## 4) Consolidación de migraciones

Se consolidaron migraciones `ALTER` en sus `CREATE` para reducir deuda técnica y simplificar bootstrap de ambientes.

- Resultado neto de consolidación: **34 → 25 migraciones**.
- Se eliminaron alteraciones dispersas de `users`, `roles`, `organization`, `employees`, `communications` y `scheduling`.
